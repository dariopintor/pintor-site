GitHub Desktop: versionamento visual

Uma interface amigável para quem está aprendendo a versionar o código sem precisar decorar comandos

Se os comandos de terminal parecem um pouco abstratos, uma alternativa é o GitHub Desktop: uma ferramenta oficial e gratuita que transforma tudo o que aprendemos sobre add, commit e push em uma interface visual amigável. O resultado final é exatamente o mesmo. Seu código é versionado, histórico é mantido, branches funcionam igual. A única diferença é como você interage com a ferramenta. Ao usá-lo lado a lado com o VS Code, você ganha o melhor dos dois mundos: o poder de edição do VS Code e a clareza visual do GitHub Desktop para gerenciar suas versões.

Instalando Git Desktop

No Windows e Mac: acesse desktop.github.com e clique no botão para baixar. O instalador é simples: baixa o arquivo, clica, e pronto. Leva 2 minutos.

No Linux: Linux é um pouco mais complicado (porque Git Desktop é feito principalmente para Windows e Mac). Mas existe. Pesquise "Git Desktop Linux" se precisar. Ou, nesse caso, pode ser que você prefira ficar com a linha de comando mesmo.

Depois de instalar, abra Git Desktop. Você verá uma interface limpa com várias opções.

Sua Primeira Vez: Configurando Git Desktop

Quando você abre Git Desktop pela primeira vez, ele pede para você se configurar. Você coloca seu nome e seu email. Isso é importante porque esses dados vão aparecer em cada commit que você fizer. Coloque o mesmo nome e email que você usa no GitHub (no futuro, quando conectar).

Criando Seu Primeiro Repositório

Diferente de usar git init na linha de comando, no Git Desktop você pode criar um repositório de forma visual.

Opção 1: Criar Novo Repositório. Clique em "File" → "New Repository". Aparece uma janela pedindo: Name (nome do repositório, tipo "meu-primeiro-projeto"), Local Path (onde você quer criar a pasta), Initialize this repository with a README (se marcar, Git cria um arquivo README automaticamente) e Git ignore (qual tipo de projeto). Clique "Create Repository" e pronto!

Opção 2: Abrir um Repositório Existente. Se você já tinha um repositório criado com git init na linha de comando, clique em "File" → "Open Repository" e escolha a pasta. Git Desktop reconhece automaticamente que é um repositório Git e carrega tudo.

Git Desktop e Sua IDE Favorita

Aqui está uma das maiores vantagens do Git Desktop: você pode estar desenvolvendo em uma IDE (VS Code, NetBeans, Xcode, IntelliJ) e depois simplesmente abrir Git Desktop para gerenciar o versionamento do mesmo projeto.

Imagine que você começou um projeto no VS Code. Criou alguns arquivos, começou a programar, tudo lindo. Mas você esqueceu de inicializar Git no começo — coisa que iniciante faz o tempo todo. Sem Git Desktop, você teria que abrir o terminal, navegar até a pasta, fazer git init, e depois configurar tudo manualmente. Com Git Desktop, é trivial:

  • Abre Git Desktop
  • Clica em "File" → "Add Local Repository"
  • Escolhe a pasta do seu projeto
  • Git Desktop pergunta: "Essa pasta não tem um repositório Git. Quer que eu crie um?"
  • Você clica "Create Repository" e pronto!

Agora o mesmo projeto que está sendo desenvolvido no VS Code está automaticamente rastreado por Git Desktop. Você continua programando no VS Code, e sempre que quer fazer um commit, abre Git Desktop, prepara os arquivos, escreve a mensagem, e commita. A beleza disso é que você trabalha onde sempre trabalhou, sem mudar sua rotina.

Se você usa VS Code, toda vez que você faz uma mudança no seu código, VS Code marca o arquivo como modificado. Git Desktop automaticamente vê isso também e mostra na tela. Você vê em tempo real o que mudou, onde mudou, quantas linhas foram adicionadas ou removidas.

Entendendo a Tela Principal

Agora você está na tela principal do Git Desktop. Pode parecer complicado no começo, mas é bem simples. Na esquerda você vê: Current Repository (nome do projeto que você está trabalhando), Branches (lista de branches — você clica para trocar) e History (histórico de todos os commits). No meio e direita você vê: Changes (arquivos que mudaram desde o último commit), Commit Summary (onde você escreve a mensagem) e o botão de Commit.

Fazendo Mudanças e Commits (Visualmente)

Você editou um arquivo no seu editor de código. Git Desktop automaticamente vê que mudou. Aqui estão os passos:

Passo 1 — Ver o que mudou. Na aba "Changes", Git Desktop mostra todos os arquivos que você alterou. Clique em um arquivo para ver exatamente o que mudou. Linhas vermelhas foram removidas. Linhas verdes foram adicionadas.

Passo 2 — Preparar os Arquivos (Stage). Na linha de comando você fazia git add. No Git Desktop é visual: cada arquivo tem um checkbox ao lado. Se quer preparar um arquivo, clica no checkbox. Se quer preparar todos, há um botão "Stage All".

Passo 3 — Escrever a Mensagem. Na caixa "Commit Summary", você escreve sua mensagem de commit — igual ao git commit -m "mensagem". Se quiser adicionar uma descrição mais longa, tem um campo "Description" abaixo.

Passo 4 — Fazer Commit. Clique no botão "Commit to [nome-da-branch]" e pronto!

Vendo o Histórico

Clique na aba "History" no lado esquerdo. Git Desktop mostra toda a timeline dos seus commits — o nome, quem fez, quando, quantas linhas foram adicionadas ou removidas. Se você clica em um commit específico, Git Desktop mostra exatamente quais arquivos mudaram e o que foi alterado linha por linha. É bem mais visual que fazer git log na linha de comando.

Branches de Uma Forma Visual

Você lembra que no Git puro você fazia git checkout -b feature/nova-funcionalidade para criar um branch? No Git Desktop é um clique.

Criando um Branch: No lado esquerdo, clique em "New Branch". Aparece uma janela pedindo o nome. Você digita, clica "Create Branch", e pronto! Git Desktop cria o branch e já muda para ele.

Trocando de Branch: Clique no nome do branch que você quer e Git Desktop faz a troca instantaneamente.

Mesclando Branches: Quando terminou de trabalhar em um branch, vá em "Branch" → "Merge into [branch-de-destino]". Git Desktop cuida de tudo. Se não tiver conflito, mescla direto. Se tiver, Git Desktop mostra qual é e te ajuda a resolver.

Enviando e Recebendo (Push e Pull)

No Git Desktop, são dois botões no topo da janela. Se você tem commits que ainda não enviou, aparece um botão "Push" no canto superior direito. Se tem mudanças no servidor que você não tem no seu computador, aparece um botão "Pull". Git Desktop também mostra quantos commits você tem para enviar e quantos tem para receber.

Clonando um Repositório do GitHub

Clique em "File" → "Clone Repository". Git Desktop abre uma janela pedindo a URL do repositório e o Local Path (onde criar a pasta). Git Desktop faz tudo sozinho: baixa o repositório inteiro, cria a pasta, inicializa Git.

Resolvendo Conflitos Visualmente

Às vezes, quando você faz merge ou pull, pode ter conflito. Na linha de comando, isso é assustador — você precisa editar o arquivo e marcar qual versão quer. No Git Desktop, é muito mais simples: ele mostra o conflito de forma visual, você vê as duas versões lado a lado e clica qual quer manter.

Voltando a um Commit Anterior

Algo deu errado? Você quer voltar para uma versão anterior do seu código? Na linha de comando, era complicado com git revert ou git reset. No Git Desktop, você clica no commit que quer no histórico e tem a opção "Revert this commit". Um clique e Git Desktop desfaz aquele commit.

Git Desktop vs Linha de Comando

Git Desktop é bom para iniciantes que se assustam com terminal, quando você quer ver visualmente o que mudou antes de commitar, quando você quer trabalhar com branches e merges sem medo, e quando você quer evitar erros. A linha de comando é boa para quando você quer ser rápido (digitando é mais rápido que clicar), quando você está em um servidor remoto sem interface gráfica, ou quando você quer fazer coisas mais avançadas.

A verdade é que não precisa escolher um. Você pode usar Git Desktop 90% do tempo e, quando precisar de algo mais específico, abrir o terminal. Ambos funcionam no mesmo repositório. Git não se importa como você o usa.

Seu Fluxo de Trabalho Com Git Desktop

Agora seu fluxo é bem mais visual. Você edita seus arquivos no editor, abre Git Desktop e vê quais arquivos mudaram, clica para preparar os arquivos, escreve a mensagem do commit, clica "Commit". Se quer enviar para o servidor, clica "Push". Se quer trazer mudanças do servidor, clica "Pull". Se quer criar um branch, clica "New Branch". Se quer ver o histórico, clica em "History". Tudo visualmente. Tudo clicando.

Próximos passos

Agora você sabe versionamento de duas formas: pela linha de comando e pela interface gráfica. Se você é iniciante e quer começar agora, comece com Git Desktop. Depois, quando estiver confortável, explore a linha de comando. Conhecer ambos torna você um desenvolvedor muito mais completo.
