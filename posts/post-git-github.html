O que é o Versionamento de Código?

Entendendo o básico do Git e GitHub

Se você está começando a programar, provavelmente já viveu aquele momento incômodo: fez uma mudança no código, algo deu errado, e então precisa voltar a uma versão anterior em que embora não estivesse muito boa estava funcionando. Outra situação parecida é quando estamos trabalhando com um colega no mesmo arquivo e no final ninguém sabe exatamente quem alterou o quê. Essas situações são tão comuns que existe uma solução inteira pensada para evitá-las. Vamos conversar sobre versionamento de código.

O versionamento de código é uma máquina do tempo para seus projetos de programação. Cada mudança que você faz fica registrada, com data, hora, e uma anotação explicando o que você alterou e por quê. Se algo der errado, você pode voltar três mudanças atrás sem problema nenhum. Se você está trabalhando com outras pessoas, cada um trabalha no seu próprio espaço e depois tudo se encaixa perfeitamente.

Esse sistema rastreia cada mudança feita em seus arquivos de programação. Funciona assim: você tem uma pasta com seus arquivos Python, JavaScript ou qualquer outra linguagem. Sempre que você faz uma mudança que considera importante, você marca aquele momento. Você escreve uma mensagem curta explicando o que mudou (algo como "Corrigi o bug da calculadora" ou "Adicionei a função de login"). O sistema guarda aquilo como um ponto na história do seu projeto.

Git e GitHub: Dois Conceitos, Uma Relação Simples

Existem várias aplicações disponíveis para versionamento de código, mas nesse artigo vamos focar no Git e no GitHub. As pessoas usam "Git" e "GitHub" como se fossem a mesma coisa, mas não são. O Git é uma ferramenta. Você baixa ela, instala no seu computador, e ela funciona localmente. Depois de aprender alguns conceitos básicos já pode versionar o seu código. Embora usar o git por terminal seja o mais comum, ele existe também na versão com interface gráfica chamada de GitHub Desktop.

Já o GitHub é um servidor na internet. Você usa o Git localmente no seu computador para fazer o trabalho de versionamento, e depois você envia tudo para o GitHub para que outras pessoas vejam, para que você tenha backup, ou para que você acesse de outro computador. O GitHub é como um Google Drive, mas feito especificamente para código.

Resumo rápido

Git é a ferramenta que você instala e usa. GitHub é o lugar onde você hospeda seus projetos Git na nuvem. Você pode usar Git sem nunca usar GitHub, mas a maioria das pessoas usa os dois juntos porque é muito mais poderoso assim.

Duas formas de usar o Git: terminal e interface gráfica

Antes de começar a utilizar o GitHub, é fundamental que você domine o Git. Existem duas formas principais de utilizá-lo: via linha de comando (direto no terminal — git-scm.com/download) ou através de uma interface gráfica, como o GitHub Desktop (desktop.github.com/download). Os conceitos do Git estão presentes em ambas as abordagens, mas na interface gráfica os comandos são executados nos bastidores.

Vantagens e Desvantagens

A principal vantagem do terminal é a velocidade e o controle total que ele oferece, além de ser universal — você vai precisar dele se for acessar um servidor diretamente. A desvantagem é a curva de aprendizado inicial, pois exige memorizar os comandos. Já a interface gráfica brilha ao mostrar o histórico de código visualmente e facilitar a compreensão de alterações, mas tem como ponto negativo a possibilidade de te deixar dependente, limitando o uso de recursos mais avançados do Git.

O que o mercado usa?

No dia a dia das empresas, o terminal é, de longe, o mais utilizado e exigido. A grande maioria das equipes de desenvolvimento espera que você saiba se virar na linha de comando. No entanto, o cenário mais comum na vida real é uma abordagem híbrida: os programadores usam o terminal para a rotina rápida (adicionar, salvar e enviar código) e recorrem a interfaces gráficas — geralmente aquelas já integradas ao próprio editor de código, como o VS Code — para tarefas que exigem muita visualização, como analisar o histórico de alterações ou resolver conflitos de código complexos.
